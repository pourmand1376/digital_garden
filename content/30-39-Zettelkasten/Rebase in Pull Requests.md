You should rebase your git history if you want clean history. At least for the branches you work and no body else works on until merged into master, you should do that. 

This way code review is also much much easier. 

[Merging vs. Rebasing \| Atlassian Git Tutorial](https://www.atlassian.com/git/tutorials/merging-vs-rebasing)
